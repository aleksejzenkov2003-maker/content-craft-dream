

## Проблема

При импорте файла «Ролики.xlsx» в раздел Вопросов, dropdown-список полей базы данных содержит только 11 полей из `QUESTION_FIELD_DEFINITIONS`, и не включает колонки файла вроде «ID ролика», «Духовник», «Ответ духовника», «Заголовок видео», «Промт для ответа». Пользователь не может привязать эти колонки к полям БД.

Корень проблемы: и Вопросы, и Ролики хранятся в одной таблице `videos`, но field definitions для вопросов содержат лишь 11 полей, а для роликов — 22. При этом некоторые поля (например `playlist_rus`) не существуют в БД как колонки.

## Что нужно изменить

### 1. `importConfigs.ts` — обновить маппинги и field definitions

**QUESTION_COLUMN_MAPPING** — добавить недостающие алиасы:
- `'id ролика'` → `video_number`
- `'духовник'` → `advisor_name` 
- `'ответ духовника'` → `advisor_answer`
- `'промт для ответа'` → `answer_prompt`
- `'заголовок видео'` → `video_title`
- `'сцены для плейлистов'` → `scene_prompt` (промт сцены, информационное поле)

**QUESTION_FIELD_DEFINITIONS** — расширить до полного набора полей таблицы `videos`, чтобы при импорте вопросов пользователь мог выбрать любое поле:
- Добавить: `video_number` (ID Ролика), `advisor_name` (Духовник), `advisor_answer` (Ответ духовника), `answer_prompt` (Промт для ответа), `video_title` (Заголовок видео), `playlist_name` (Плейлист — резолвится в `playlist_id`)
- Убрать несуществующее поле `playlist_rus` (в БД нет такой колонки — плейлисты привязываются через `playlist_id`)

**VIDEO_COLUMN_MAPPING** — добавить алиасы:
- `'промт для ответа'` → `answer_prompt`
- `'сцены для плейлистов'` → игнорировать (это из другой таблицы)

**VIDEO_FIELD_DEFINITIONS** — добавить недостающие:
- `answer_prompt` (Промт для ответа)
- Убедиться что все поля из image-56 присутствуют

### 2. `QuestionsTable.tsx` — обновить трансформацию при импорте

Текущий `onImport` handler обрабатывает только 8 полей. Нужно добавить:
- `video_number` → parseInt
- `advisor_answer` → String
- `answer_prompt` → String  
- `video_title` → String
- `playlist_name` → резолвить в `playlist_id` по имени из списка `playlists`

### 3. `QuestionsTable.tsx` — передать `resolveRow` в CsvImporter

Для резолва `advisor_name` → `advisor_id` и `playlist_name` → `playlist_id` нужно добавить `resolveRow` проп в CsvImporter (он уже поддерживается компонентом), чтобы при маппинге "Духовник" и "Плейлист" корректно резолвились в ID из справочников.

### Итог

После изменений:
- Dropdown в импорте вопросов будет содержать все поля таблицы `videos` (ID Ролика, Духовник, Плейлист, Ответ духовника, Промт для ответа, Заголовок видео и т.д.)
- Поле `playlist_rus` будет убрано (не существует в БД)
- Все колонки из CSV файла смогут быть привязаны к реальным полям БД
