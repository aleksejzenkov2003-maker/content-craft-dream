

## Диагностика

Проблема найдена в функции `mergeStsd` (строки 277-285 в `concat-video/index.ts`). Текущая логика **всегда** дедуплицирует аудио `stsd`, используя только описание кодека из **первого** файла (основного видео). Это означает, что все аудио-семплы второго файла (задней обложки) декодируются с настройками первого файла. Если параметры кодека отличаются (sample rate, AAC profile, channel config) — звук второго файла воспроизводится как тишина.

В логах видно: `Track soun: deduped identical stsd entries for decoder compatibility` — но на самом деле записи могут быть **не идентичны**.

## План исправления

### Изменение в `supabase/functions/concat-video/index.ts`

1. **Сравнивать stsd-записи побайтово** перед дедупликацией. Дедуплицировать только если записи действительно идентичны (побайтовое совпадение).

2. **Если записи различаются** — включать обе в merged stsd и корректно маппить `sdi` для второго файла (как уже делается для видео-трека). Добавить диагностическое логирование с деталями различий (длина записей, первые байты).

3. **Добавить логирование** ключевых параметров аудио (sample rate из stsd entry) для отладки будущих проблем.

### Техническая суть

```
// Вместо безусловного dedup:
if (handlerType === "soun" && entryCount1 === 1 && entryCount2 === 1) {
  // Сравниваем байты entry1 и entry2
  if (bytesEqual(entries1[0], entries2[0])) {
    // Действительно идентичны → dedup безопасен
    return { merged: buildStsdFromEntries(entries1), mapSdiFile2: () => 1, ... };
  }
  // Различаются → merge обе записи, как для видео
  console.log("Audio stsd entries differ, merging both");
}
```
