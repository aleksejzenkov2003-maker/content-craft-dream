
# План разработки интерфейса платформы

## Обзор

План разбит на **5 фаз** по приоритету реализации. Каждая фаза содержит логически связанные задачи, которые можно выполнять параллельно внутри фазы.

## Фаза 1: UI-система и базовые компоненты (3-4 дня)

### 1.1 Унифицированная система кнопок
**Цель:** Создать единый компонент кнопок для всего приложения

**Задачи:**
- Расширить `src/components/ui/button.tsx` новыми вариантами
- Добавить варианты: `generate`, `publish`, `delete-inline`, `icon-action`
- Создать цветовые схемы: orange (generate cover), purple (generate video), blue (publish)
- Унифицировать размеры: `xs`, `sm`, `md` для таблиц
- Заменить кастомные стили в `VideosTable`, `PublicationsTable`, `ScenesMatrix`

**Файлы:**
- `src/components/ui/button.tsx` - расширение вариантов
- Все таблицы - замена на унифицированные кнопки

### 1.2 Компонент inline-редактирования
**Цель:** Создать переиспользуемый компонент для редактирования прямо в ячейке таблицы

**Задачи:**
- Создать `InlineEdit` компонент (клик → input/select → blur сохраняет)
- Поддержка типов: text, date, datetime, select, status
- Валидация и обратная связь при ошибках
- Keyboard shortcuts (Enter - сохранить, Esc - отмена)

**Файлы:**
- `src/components/ui/inline-edit.tsx` - новый компонент

### 1.3 Компонент массовых действий
**Цель:** Унифицировать панель массовых действий для всех таблиц

**Задачи:**
- Создать `BulkActionsBar` компонент
- Чекбоксы для выбора строк (уже частично есть в QuestionsTable)
- Отображение счётчика выбранных
- Слоты для кастомных кнопок действий

**Файлы:**
- `src/components/ui/bulk-actions-bar.tsx` - новый компонент

### 1.4 Компонент изменяемой ширины столбцов
**Цель:** Добавить возможность изменения ширины столбцов перетаскиванием

**Задачи:**

**Файлы:**

## Технические детали

### Изменения в базе данных

```text
1. video_variants (новая таблица)
   - id: uuid
   - video_id: uuid (FK → videos)
   - type: text ('cover' | 'video')
   - url: text
   - prompt: text
   - is_selected: boolean
   - created_at: timestamp

2. publications - уникальный индекс
   - UNIQUE (video_id, channel_id)

3. scene_variants (новая таблица) ИЛИ расширение playlist_scenes
   - id: uuid
   - playlist_scene_id: uuid
   - scene_url: text
   - prompt: text
   - is_selected: boolean
   - created_at: timestamp

4. publishing_channels - новое поле
   - back_cover_url: text
```

### Новые компоненты

```text
src/components/ui/
├── inline-edit.tsx          # Inline редактирование
├── bulk-actions-bar.tsx     # Панель массовых действий
├── resizable-table.tsx      # Изменяемая ширина столбцов
├── media-preview.tsx        # Превью медиа с lightbox
└── channel-selector.tsx     # Мультиселект каналов

src/hooks/
├── useVideoVariants.ts      # Варианты генерации
└── useSceneVariants.ts      # Варианты сцен
```

### Порядок реализации

```text
Неделя 1:
├── Фаза 1.1: Унификация кнопок
├── Фаза 1.2: InlineEdit компонент

Неделя 2:

Неделя 3:

└── Фаза 3.4-3.6: Ролики (промпты, варианты, UI)

Неделя 4:
├── Фаза 4.1-4.3: Публикации (генерация, каналы, редактирование)
```

## Приоритеты

**Высокий приоритет (критично для работы):**
1. Inline-редактирование даты/статуса в Вопросах
2. Массовые действия (чекбоксы, удаление, смена статуса)
3. Фильтры и сортировка в таблицах

**Средний приоритет (улучшение UX):**
1. Унификация кнопок
2. Встроенные плееры
3. Редактируемые промпты
4. Выбор каналов чекбоксами

**Низкий приоритет (nice to have):**
1. Мультивариантность генерации
2. Изменяемая ширина столбцов
3. Привязка обложек к каналам
4. Склейка видео с задней обложкой
