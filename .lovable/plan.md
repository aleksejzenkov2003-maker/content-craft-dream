
# План: Исправление группировки вопросов и роликов

## Выявленная проблема

Анализ базы данных показал, что **один question_id соответствует разным текстам вопросов**:

| question_id | question_rus | Кол-во роликов |
|-------------|--------------|----------------|
| 2 | Можно ли разводиться? | 7 |
| 2 | Что делать, если я не чувствую присутствие Бога... | 5 |
| 3 | Как молиться правильно? | 6 |
| 3 | Как сделать молитву ежедневной привычкой? | 6 |
| 9 | В чём разница между молитвой и медитацией? | 5 |
| 9 | Как исповедоваться? | 7 |

Это приводит к тому, что:
- В QuestionsTable группировка по `question_id` показывает только **первый попавшийся текст** (5 записей)
- При обновлении статуса по `question_id` обновляются **все ролики** с этим ID, независимо от текста
- В VideosTable группировка идёт по **тексту вопроса**, поэтому показываются все 7+ уникальных текстов

## Решение

Использовать **составной ключ** для идентификации уникального вопроса: `question_id + question_rus`.

### Изменения в QuestionsTable

**Файл:** `src/components/questions/QuestionsTable.tsx`

1. Изменить тип ключа группировки с `number` на `string` (составной ключ)
2. Обновить интерфейс `QuestionData` - добавить `unique_key: string`
3. Изменить логику агрегации:

```typescript
const uniqueKey = `${video.question_id}_${video.question_rus || video.question_eng || ''}`;
const existing = questionMap.get(uniqueKey);
// ...
questionMap.set(uniqueKey, {
  unique_key: uniqueKey,
  question_id: video.question_id,
  // остальные поля
});
```

4. Обновить все места где используется `question_id` для:
   - Отображения строк (key в map)
   - Выбора для bulk actions
   - Фильтрации

### Изменения в VideosTable

**Файл:** `src/components/videos/VideosTable.tsx`

1. Добавить отображение `question_id` в заголовке группы:
   - Показывать: "ID: {question_id} | {question_rus}"
   
2. Группировать по составному ключу вместо текста вопроса

### Изменения в Index.tsx

**Файл:** `src/pages/Index.tsx`

1. Обновить `onUpdateQuestion` - искать ролики по составному ключу (question_id + question_rus)
2. Обновить bulk операции для корректной идентификации

## Технические детали

| Файл | Изменение |
|------|-----------|
| `src/components/questions/QuestionsTable.tsx` | Группировка по составному ключу, обновление интерфейса QuestionData |
| `src/components/videos/VideosTable.tsx` | Отображение question_id в заголовке группы, группировка по составному ключу |
| `src/pages/Index.tsx` | Обновление логики обновления/удаления по составному ключу |

## Альтернативное решение (чистка данных)

Если дублирование question_id - это ошибка данных, можно:
1. Исправить данные в базе - присвоить уникальные question_id каждому уникальному вопросу
2. Оставить текущий код без изменений

Рекомендуется **первый вариант** (составной ключ), так как он не требует ручного исправления данных и предотвращает подобные проблемы в будущем.
