

## План: Полная матрица автоматизации (все кнопки × все процессы)

### Проблема
Сейчас в таблице `automation_settings` только 27 записей — покрыты лишь те комбинации, которые были явно добавлены. Пользователь хочет видеть **полную матрицу** — чекбокс на каждом пересечении кнопки и процесса, чтобы настраивать весь бизнес-процесс гибко.

### Решение

#### 1. Миграция БД: заполнить все комбинации

7 колонок (button_key) × 11 процессов (process_key) = 77 ячеек. Сейчас 27 — нужно добавить ~50 записей.

SQL с `INSERT ... ON CONFLICT DO NOTHING` добавит все недостающие пары. Новые записи будут `is_enabled = false` по умолчанию (не ломает текущее поведение). Существующие записи не изменятся.

Кнопки: `take_in_work`, `side_step1`, `side_cover`, `generate_video`, `bulk_generate_covers`, `side_video`, `prepare_publish`, `bulk_publish`, `publish`

Процессы: `voiceover`, `atmosphere`, `auto_atmosphere`, `cover_overlay`, `heygen`, `resize`, `subtitles`, `concat`, `create_publication`, `generate_text`, `publish_social`

#### 2. Обновить `ButtonActionsSettings.tsx`

- Добавить колонку `side_cover` (Шаг 2. Обложка) — сейчас в COLUMNS уже есть, но в БД мало записей
- Убедиться что все 7 колонок и 11 процессов отображаются
- Каждая ячейка теперь будет иметь чекбокс (не «—»), т.к. запись существует в БД

#### 3. Код `Index.tsx` — без изменений

Все `isEnabled()` проверки уже на месте. Новые записи с `is_enabled = false` не меняют поведения. Когда пользователь включит чекбокс — процесс активируется автоматически.

### Итог
Полная матрица 9×11 = 99 чекбоксов (с учётом `bulk_generate_covers` и `bulk_publish` как отдельных button_key). Каждый управляем.
