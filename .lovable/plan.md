

## Исправление: кнопка "Шаг 1. ФОН" запускает только назначенные процессы

### Проблема

При клике на "Шаг 1. ФОН" обложка тоже переходит в статус "In progress", хотя в настройках автоматизации `cover_overlay` для `side_step1` выключен.

### Причины

1. **Edge-функция `generate-cover`** (строка 287) безусловно ставит `cover_status: 'generating'` ещё до определения, какие шаги нужно выполнить. Даже при `step === 'atmosphere'` обложка помечается как "генерируется".

2. **`useAutomationSettings` hook** — функция `isEnabled()` возвращает `true` по умолчанию, если настройки ещё не загрузились. Это может вызвать ложное срабатывание автосклейки обложки.

### Исправления

#### 1. Edge-функция `generate-cover/index.ts`

Перенести `cover_status: 'generating'` после определения шагов. Ставить этот статус только если `runOverlay === true`. Если только атмосфера — ставить отдельный статус (например, `atmosphere_generating`).

#### 2. Hook `useAutomationSettings.ts`

Изменить дефолт `isEnabled()` с `true` на `false` пока данные не загружены — чтобы при загрузке приложения ни один авто-процесс не запускался случайно. Добавить проверку `isLoading`.

#### 3. `Index.tsx` — `handleGenerateAtmosphere`

Без изменений — `isEnabled('side_step1', 'cover_overlay')` уже проверяется корректно, после фикса хука будет работать правильно.

### Итог

- "Шаг 1. ФОН" генерирует только фон, без побочного эффекта на обложку
- Автосклейка обложки сработает только если чекбокс включён в матрице и данные загружены
