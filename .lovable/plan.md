
## План: Исправить данные — добавить ролики для всех духовников

### Текущая ситуация
- В базе **13 активных духовников**
- На каждый вопрос существует только **1-2 видео** вместо 13
- Код автоматического создания видео для всех духовников уже есть (Index.tsx строки 333-347), но старые данные были созданы до этого изменения

### Что нужно сделать

**Вариант 1: Одноразовое обновление данных (SQL)**

Добавить недостающие видео через SQL запрос — для каждого существующего вопроса создать записи для всех духовников, которых ещё нет.

```sql
INSERT INTO videos (question_id, question, question_rus, safety_score, advisor_id, video_number)
SELECT DISTINCT 
  v.question_id,
  v.question,
  v.question_rus,
  v.safety_score,
  a.id as advisor_id,
  v.question_id * 100 + (ROW_NUMBER() OVER (PARTITION BY v.question_id ORDER BY a.name))
FROM videos v
CROSS JOIN advisors a
WHERE a.is_active = true
  AND NOT EXISTS (
    SELECT 1 FROM videos v2 
    WHERE v2.question_id = v.question_id 
      AND v2.advisor_id = a.id
  );
```

**Вариант 2: Кнопка "Синхронизировать" в UI**

Добавить кнопку в интерфейсе вопросов, которая проверит все вопросы и создаст недостающие видео для духовников.

### Рекомендация

Использую **Вариант 1** — одноразовый SQL запрос для исправления существующих данных. Это быстрее и надёжнее.

### Результат
- Каждый вопрос будет иметь по 13 видео (по одному на каждого духовника)
- В таблице роликов под каждым вопросом будет отображаться список всех духовников как на референсном изображении
