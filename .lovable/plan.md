

## Проблема склейки видео: таймаут загрузки ffmpeg.wasm

### Корень проблемы

`ffmpeg.wasm` — это ~30MB WASM-файл, загружаемый с CDN `unpkg.com`. Таймаут стоит 30 секунд — этого часто недостаточно для загрузки такого объёма, особенно при нестабильном соединении.

### План исправлений

#### 1. Перенести склейку на сервер (Edge Function `concat-video`)

Вместо загрузки ffmpeg.wasm в браузер — делать склейку на сервере. Edge Function `concat-video` уже существует, но только обновляет БД. Переделать её:

- Принимает `publication_id`, `main_video_url`, `back_cover_video_url`
- Скачивает оба видео через `fetch()` (на сервере нет CORS проблем)
- Склеивает MP4 файлы через конкатенацию на уровне байтов (для совместимых MP4 — это простая операция с `ftyp`/`moov`/`mdat` боксами), либо использует библиотеку `mp4box` для Deno
- Загружает результат в Storage и обновляет `publications`

**Упрощённый подход**: Если видео имеют одинаковые кодеки (что верно для HeyGen MP4 + back cover MP4), можно использовать простую бинарную конкатенацию через `mp4-muxer` или `mp4box.js`. Однако самый надёжный путь — использовать ffmpeg через Deno WASM (пакет `ffmpeg-wasm` для Deno) на сервере, где нет ограничений по таймауту CDN.

**Реалистичный подход**: Использовать npm-пакет `mp4-muxer` (доступен через esm.sh) для демуксинга и ремуксинга двух MP4 в один на сервере. Это легковесно и не требует ffmpeg.

#### 2. Обновить клиентский хук `useVideoConcat`

- Убрать всю логику ffmpeg.wasm
- Заменить на вызов Edge Function: `supabase.functions.invoke('concat-video', { body: { publication_id, main_video_url, back_cover_video_url } })`
- Поллить статус публикации до завершения

#### 3. Файлы для изменения

- `supabase/functions/concat-video/index.ts` — полная переработка: скачивание видео, конкатенация, загрузка в Storage
- `src/hooks/useVideoConcat.ts` — упростить до вызова Edge Function + polling
- `supabase/functions/proxy-video/index.ts` — больше не нужен (удалить)
- `supabase/config.toml` — добавить `[functions.concat-video]` с `verify_jwt = false`

### Результат

Склейка происходит на сервере без загрузки 30MB WASM в браузер. Нет таймаутов CDN, нет CORS проблем, нет зависимости от скорости интернета пользователя.
